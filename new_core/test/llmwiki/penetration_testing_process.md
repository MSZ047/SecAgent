# 渗透测试流程

## 概述
本文档记录完整渗透测试流程，包含信息收集、漏洞发现、漏洞利用、权限维持等阶段。


## 测试阶段

### 1. 信息收集
- 获取本机IP地址：`ifconfig`
- 计算网段：IP地址与子网掩码进行与运算
- 端口扫描：`nmap -sP 192.168.56.0/24`
- 服务发现：`nmap -A -T4 <target>`
- 目录扫描：`dirb <target>`

### 2. 漏洞发现
- 识别Web应用
- 发现可利用点（如文件上传点、登录口）

### 3. 漏洞利用
- 横向越权：抓包修改id参数
- 文件上传：上传Webshell

### 4. 获取shell
- 蚁剑连接
- 反向监听
- 获取Meterpreter

### 5. 提权
- 环境变量劫持
- SUID滥用
- Sudo权限滥用

## Web服务器运行原理

### 架构组成
1. **硬件层**：CPU、内存、存储
2. **系统层**：Linux/Windows
3. **网络层**：IP地址配置，确保与攻击机同网段
4. **服务层**：Web服务软件（Apache/Nginx）+ 脚本解析器（PHP）
5. **应用层**：网站代码和目录

### 请求流程
1. 用户浏览器发送HTTP请求
2. Web服务软件（80端口）接收请求
3. 脚本解析器（PHP）执行网站代码
4. 代码调用数据库获取数据
5. 结果通过HTTP响应返回用户

## 常见Web漏洞

### SQL注入
- 原理：在输入框注入恶意SQL语句
- 示例：`or 1=1`
- 防御：使用预编译语句

### XSS跨站脚本
- 原理：注入恶意脚本
- 示例：`<script>alert('xss')</script>`
- 类型：反射型、存储型、DOM型
- 防御：输出编码

## IP与域名关系
- 域名是IP地址的别名
- 通过DNS解析将域名转换为IP地址
- 访问结果一致

## 前端与后端区别
- **前端**：用户界面，运行在浏览器，容易被修改
- **后端**：数据处理和逻辑判断，多种编程语言实现
- **通信**：通过HTTP请求/响应进行数据交换
