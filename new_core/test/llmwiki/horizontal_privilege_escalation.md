# 横向越权测试

## 概述
横向越权是指攻击者通过篡改身份标识，以普通用户身份执行管理员操作的安全漏洞。

## 漏洞原理
网站后端根据ID值辨别用户身份，如果前端传递的ID参数未做严格校验，攻击者可通过修改参数值来冒充其他用户。

## 攻击示例

### 抓包发现ID
1. 注册两个账号
2. 抓包对比数据，发现id参数随账号递增
3. 常见规律：管理员id=1，后续用户id=2,3...

### 改包攻击
1. 使用Burp Suite拦截请求包
2. 修改id参数值为1
3. 修改password为指定值
4. 转发数据包

## 防御策略

### 不暴露敏感标识
- 不在URL中明文传递用户id
- 不在表单隐藏域中传递权限等级

### 服务端全权负责
- ID的生成与验证由服务端完成
- 前端仅作展示，不参与逻辑判定

### 传输加密
- 全程启用HTTPS加密
- 杜绝中间人篡改风险

### 身份判定增强
- **JWT身份判定**：采用强加密算法签名，密钥定期轮换
- **Cookie身份判定**：设置HttpOnly禁止JS读取，防止XSS窃取
